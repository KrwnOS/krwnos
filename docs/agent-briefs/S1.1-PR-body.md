## What the tests exercise

**`service.test.ts`** — describe blocks:

- `getAccessibleBoards` (7): unauthenticated → `UNAUTHORIZED`; global reader sees global boards; node reader sees only their own node's boards; mixed access (no global read, some node reads) returns node boards without throwing; no read anywhere → `FORBIDDEN`; owner sees everything; empty state returns `[]`.
- `getBoardById` (5): unauthenticated → `UNAUTHORIZED`; missing board → `NOT_FOUND`; global-scoped board gated by global `tasks.read`; node-scoped board gated by `canOnNode`; owner bypass.
- `createBoard` (4): unauthenticated → `UNAUTHORIZED`; lacks `tasks.admin` → `FORBIDDEN`; node admin creates + emits `core.tasks.board.created`; owner creates global board without explicit grant.
- `createTask` (5): unauthenticated; missing board → `NOT_FOUND`; no `tasks.write` → `FORBIDDEN`; writer creates + emits `core.tasks.task.created`; owner bypass.
- `updateTask` (5): mirrors `createTask` and asserts the emitted `updates` payload equals the keys the caller passed.
- `error shape` (2): every refusal is a `KrwnError` instance with the documented `code` field.
- `manifest` (3): `coreTasksModule.init()` returns the descriptors, `getWidget()` is guarded by `tasks.read`, `getSettings()` returns `null`.

**`repo.test.ts`** — pinned Prisma call shapes for: `getBoards`, `getBoardById`, `createBoard` (seeds the three default columns), `createColumn` (max-order + 1, and empty fallback to 0), `updateColumnOrder` (sibling reshuffle via `updateMany`), `createTask`, `updateTask` (same-column reorder, cross-column move shifts both columns, partial updates), `deleteTask` (collapses remaining column order).

## Not done on purpose

Both deferred to later stages per "Don't refactor service/repo to make them testable":

- **Prisma → `KrwnError` translation in `TasksRepository`.** Errors like P2025 bubble up raw; the tests document the current behaviour rather than asserting a translated shape. Likely candidate for S1.2 / S1.3.
- **Delete paths at the service layer.** `TasksService` has no `deleteBoard` or `deleteTask` method; only the repo does. Worth confirming before S1.2 assumes otherwise.
- **No CHANGELOG entry.** Tests-only change.
- **No edits to `service.ts`, `repo.ts`, `index.ts`, `permissions.ts`, or `vitest.config.mts`.** The repo-wide coverage gate is scoped to `src/core/**` and stays that way in this PR; widening the gate to include `src/modules/**` is a separate, deliberate decision.
