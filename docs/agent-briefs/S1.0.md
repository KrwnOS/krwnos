**Stage:** 1 (prep) · **Size:** S (2–4 hours) · **Blocks:** S1.1, S1.2,
S1.3, S1.4, S1.5

## Goal

Commit every currently-uncommitted change on `cursor/mission-whitepaper-obligations`
in logical, reviewable chunks, and push the branch to origin. After this task
the branch becomes a normal feature branch that subsequent stacked PRs can
target.

## Why this matters

The Stage 1 briefs (S1.1–S1.4) assumed the WIP branch already had committed
code that new tests and refactors could build on. That assumption was wrong —
the work is sitting as unstaged/untracked files in the working tree. Nothing
downstream can proceed until those changes are committed somewhere real.

This task does only that. It does not add new code, new tests, or new
documentation. It is purely a commit-and-push operation.

## Do this

1. Group the changes into logical commits. Suggested split (you may refine):
   - **Commit 1 — Module sandbox kernel work.** `src/core/module-sandbox.ts`,
     `src/core/__tests__/module-sandbox.test.ts`, the parts of
     `src/core/index.ts` that export it, and the SDK contract additions in
     `packages/sdk/src/module-contract.ts`, `packages/sdk/src/harness.ts`,
     `packages/sdk/src/index.ts`. Include the portions of
     `prisma/schema.prisma` that support module schema isolation, plus any
     matching migration files under `prisma/migrations/`.
   - **Commit 2 — `core.tasks` module.** `src/modules/tasks/**` (service,
     repo, permissions, index), the relevant part of
     `src/modules/index.ts` that registers it, and the parts of
     `prisma/schema.prisma` defining `TaskBoard` / `TaskColumn` / `Task`
     models plus any matching migration.
   - **Commit 3 — Tasks API and UI surface.** `src/app/api/tasks/**`,
     `src/app/dashboard/tasks/**`, `src/components/tasks/**`.
   - **Commit 4 — Changelog scaffolding.** `CHANGELOG.md`,
     `.cursor/rules/changelog.mdc`. (S1.4 will reconcile this against
     `ROADMAP.md §9`; for now, commit them as-is.)
   - **Commit 5 — Roadmap + dependency bumps.** `docs/ROADMAP.md`,
     `package.json`, `package-lock.json`.
2. Commit message style: match the existing repo convention. Look at
   `git log --oneline -20` for examples. Typical shape:
   `feat(core): module sandbox for third-party modules` or
   `feat(modules): core.tasks kanban service + repo`.
3. Push to origin. If the remote has a protected-branch policy, flag it in
   your report — don't bypass it.
4. Verify `git log --oneline cursor/mission-whitepaper-obligations` shows
   the expected commits in a sensible order.
5. Run `npm run typecheck`, `npm run lint`, and `npm test` on the final
   branch state. They do not all need to pass — the WIP is incomplete by
   design, which is why S1.1–S1.4 exist — but you should record which
   commands fail and with what errors in your report, so the next agent
   knows the starting baseline.

## Don't do

- Don't rebase, squash, or rewrite any existing commits on the branch. You
  are only adding new commits for the currently-uncommitted work.
- Don't write new code, add new tests, or change functionality. If a file
  has a typo and you're tempted to fix it, note it in your report instead.
  That's a separate PR.
- Don't open a PR against `main`. This branch is not yet ready to merge —
  that's S1.5.
- Don't modify `docs/agent-briefs/` or any of the S1.\*.md briefs, even if
  they look wrong. Surface any drift in your report.

## Pitfalls

- Prisma schema + migrations: if the uncommitted `prisma/schema.prisma`
  diff adds `TaskBoard`/`TaskColumn`/`Task` models but there is no
  corresponding migration file, you must run `npx prisma migrate dev
  --create-only --name core_tasks_initial` (or similar) to generate the
  migration, then include it in the appropriate commit. Check
  `prisma/migrations/` for the most recent migration first to understand
  the naming pattern.
- Line endings: the repo has mixed CRLF/LF and git warns on commit. This
  is expected; don't fight it.
- CHANGELOG.md reconciliation (S1.4) will re-touch `CHANGELOG.md` and the
  cursor rule. That's fine — committing them now doesn't pre-commit you
  to their current wording.
