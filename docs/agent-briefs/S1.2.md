# S1.2 — Extract shared `getAuthenticatedContext`

**Stage:** 1 · **Size:** S (half day) · **Blocked by:** S1.1 · **Blocks:** S1.3

## Goal

Move `getAuthenticatedContext` to a shared location so future modules don't
copy the "borrow from chat" shortcut the tasks module currently uses.

## Why this matters

`src/app/api/tasks/boards/route.ts` contains this line:

```ts
import { getAuthenticatedContext } from "@/app/api/chat/_context"; // Reusing the same auth wrapper for now
```

That "for now" is load-bearing. Every future module shipped through the
marketplace (Stages 2–7) will reach for API-route authentication, and the
current shape implies "copy whatever the last module did." A shared helper
in a neutral location becomes the canonical pattern documented in
`MODULE_GUIDE.md`.

## Starting state

- `src/app/api/chat/_context.ts` contains the current implementation.
- `src/app/api/tasks/boards/route.ts` and `src/app/api/tasks/tasks/route.ts`
  both import from chat's `_context`.
- Every other existing API route uses either its own `_context` file or
  rolls auth inline (there's no consistent pattern yet).

## Do this

1. Create `src/app/api/_shared/auth-context.ts`. Move the implementation
   from `src/app/api/chat/_context.ts` into it, keeping the public surface
   identical (same function names, same return types).
2. Leave a thin re-export shim at `src/app/api/chat/_context.ts` that
   imports from the shared location, so existing chat routes don't break.
   Mark the shim `@deprecated` with a note to migrate.
3. Update `src/app/api/tasks/boards/route.ts` and `.../tasks/route.ts` to
   import from `@/app/api/_shared/auth-context`. Remove the "for now"
   comment.
4. Add a short contract test `src/app/api/_shared/__tests__/auth-context.test.ts`
   covering: missing bearer token → 401; expired token → 401; valid token
   returns a populated `ctx`.
5. Update `docs/MODULE_GUIDE.md` with a section titled "Authentication in
   API routes" pointing to the shared helper as the canonical pattern.

## Definition of done

- [ ] `src/app/api/_shared/auth-context.ts` is the single source of truth.
- [ ] `src/app/api/chat/_context.ts` is a deprecated re-export shim.
- [ ] Both tasks API routes import from the shared location.
- [ ] Contract test passes.
- [ ] `docs/MODULE_GUIDE.md` updated.
- [ ] `npm run typecheck`, `npm run lint`, `npm test`, e2e smoke all green.

## Don't do

- Don't delete the chat `_context.ts` shim outright. Other chat-scoped code
  may still import from it; the shim keeps the diff tight and follow-up work
  can migrate the rest later.
- Don't change the function signatures. This is a move, not a redesign. The
  next time to re-shape `getAuthenticatedContext` is when S1.3 wires tasks
  through the module sandbox.
- Don't touch routes outside chat and tasks. Other routes using their own
  `_context` can migrate opportunistically later.

## Pitfalls

- `src/app/api/chat/_context.ts` may contain chat-specific imports that
  shouldn't move to the shared file. Audit what's actually used by the
  exported helper; move only the generic parts. Anything chat-specific stays
  behind.
- Import cycles: `_shared/` should not import from any feature directory
  (`chat/`, `tasks/`, etc.). Only from `@/core/*`, `@/lib/*`, `@/types/*`.
